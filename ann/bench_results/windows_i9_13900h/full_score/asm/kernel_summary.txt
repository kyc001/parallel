Real-kernel objdump summary (g++ -O3 -mavx2 -mfma, main_win_avx2_g.exe)
Source: bench_results/windows_i9_13900h/full_score/asm/full_objdump.txt

===== Flat-AVX2 inner loop (flat_search at 0x140001a40) =====
Four 256-bit accumulators, 32 floats per iter, FMA throughput ~= 2/clock.
  vmovups ymm4,YMMWORD PTR [rdx]              ; load 8 floats (lane 0)
  vmovups ymm5,YMMWORD PTR [rdx+0x20]         ; load 8 floats (lane 1)
  sub     rdx,-128                            ; i += 128 B (32 floats)
  sub     r9,-128
  vfmadd231ps ymm0,ymm4,YMMWORD PTR [r9-0x80] ; acc0 += x0*y0
  vfmadd231ps ymm2,ymm5,YMMWORD PTR [r9-0x60] ; acc2 += x1*y1
  vmovups ymm4,YMMWORD PTR [rdx-0x40]
  vmovups ymm5,YMMWORD PTR [rdx-0x20]
  vfmadd231ps ymm1,ymm4,YMMWORD PTR [r9-0x40] ; acc1 += x2*y2
  vfmadd231ps ymm3,ymm5,YMMWORD PTR [r9-0x20] ; acc3 += x3*y3
  cmp     rdx,r10
  jne     loop_start
Observation: 4 indep FMAs, no reduction inside loop, reduction pushed to
post-loop vaddps + vhaddps. Matches the manual-AVX2 style in section 4.4.

===== SQ-AVX2 uint8 L2 inner loop (sq_search at 0x140003a40) =====
256-bit integer SIMD: 32 uint8/iter, expanded to 16-bit, dot-product via
vpmaddwd; this is the 28.6% "256-bit Int Vector uOps" VTune reports.
  vmovdqu    ymm1,[rdx+rax*1]    ; 32 uint8 from x
  vmovdqu    ymm2,[rsi+rax*1]    ; 32 uint8 from y
  add        rax,0x20
  vpunpcklbw ymm0,ymm1,ymm3      ; expand low  16 bytes -> 16 words
  vpunpcklbw ymm5,ymm2,ymm3
  vpunpckhbw ymm1,ymm1,ymm3      ; expand high 16 bytes -> 16 words
  vpunpckhbw ymm2,ymm2,ymm3
  vpsubw     ymm0,ymm0,ymm5      ; diff (lo)
  vpsubw     ymm1,ymm1,ymm2      ; diff (hi)
  vpmaddwd   ymm0,ymm0,ymm0      ; diff^2 then pairwise -> 8 dwords
  vpmaddwd   ymm1,ymm1,ymm1
  vpaddd     ymm0,ymm0,ymm1
  vpaddd     ymm0,ymm0,ymm4      ; running 32-bit accumulator
  vmovdqa    ymm4,ymm0
  cmp        rax,rbp
  jne        loop_start
Observation: 96-dim uint8 L2 collapses to 3 iters, with vpmaddwd doing
"square + pair-add" in one uOp. This is the structural reason SQ is much
less memory-bound than Flat.

===== PQ-AVX2 LUT construction inner loop (pq_search at 0x1400027e0) =====
One-accumulator 8-float FMA; each iter handles 12-dim sub-vector dot.
  vmovups ymm4,YMMWORD PTR [r8+rax*1]
  vfmadd231ps ymm0,ymm4,YMMWORD PTR [rcx+rax*1]
  add     rax,0x20
  cmp     rax,r10
  jne     loop_start
Observation: single-accumulator version; shorter latency hiding but
sufficient at dsub=12 (loop count is tiny). ADC lookup itself remains
scalar (see report 3.3 for why FastScan is needed to vectorize that).
